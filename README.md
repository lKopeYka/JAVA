##Ямонт Михаил Денисович 450501
## О проекте

Цель работы — разработать прототип информационной системы для библиотеки с использованием объектно-ориентированного подхода и реляционной базы данных. Приложение работает через командную строку и позволяет моделировать основные бизнес-процессы библиотеки: поступление книг, регистрацию читателей, выдачу и прием литературы.

## Модель данных (сущности)

В системе выделены следующие ключевые сущности:

| Сущность | Описание | Ключевые атрибуты |
|----------|----------|-------------------|
| **Book** (Книга) | Библиографическое описание издания | ID, Название, ISBN, Год издания, Жанр |
| **Author** (Автор) | Авторы книг (связь многие-ко-многим) | ID, ФИО, Дата рождения, Страна |
| **Copy** (Экземпляр) | Физический экземпляр книги на полке | Инвентарный номер, Статус, Номер полки |
| **Reader** (Читатель) | Зарегистрированный пользователь | Номер билета, ФИО, Телефон, Email, Дата регистрации |
| **Loan** (Выдача) | Операция выдачи/возврата | ID операции, Дата выдачи, Плановая дата возврата, Фактическая дата возврата, Штраф |
| **Librarian** (Библиотекарь) | Сотрудник библиотеки | ID, ФИО, Логин, Пароль (хеш), Роль |

## Функциональные возможности

### Для читателей:
- Просмотр каталога книг
- Поиск по названию, автору, жанру
- Просмотр списка выданных книг
- История чтения

### Для библиотекарей:
- **Управление книгами**: добавление, редактирование, удаление
- **Управление авторами**: добавление новых авторов, связь с книгами
- **Управление экземплярами**: добавление новых копий, изменение статуса (выдана/в наличии/реставрация)
- **Регистрация читателей**: создание и редактирование профилей
- **Операции выдачи/возврата**:
  - Оформление выдачи книги читателю
  - Оформление возврата
  - Расчет штрафов за просрочку
- **Просмотр статистики**: самые популярные книги, должники, задолженности
